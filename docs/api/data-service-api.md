# Data Service API

**Base URL:** `http://<node-host>:8769`  
**Source:** `Services/data_service/api/routes.py`, `Services/data_service/runtime.py`

The Data Service records selected parameter values from ParameterDB to files at a configurable sampling rate and supports timed loadstep averaging over the same parameter set.

---

## Measurement Lifecycle

1. Call `POST /measurement/setup` with the parameter list, sample rate, and output format.
2. Call `POST /measurement/start` to begin recording.
3. Optionally call `POST /loadstep/take` one or more times during recording.
4. Call `POST /measurement/stop` to finalize the file and return a summary.

The service requires an active ParameterDB backend connection before setup is accepted.

---

## Data Model Notes

### Measurement Configuration

```json
{
  "parameters": ["brewcan.temperature.0", "psu.meas_voltage"],
  "hz": 10.0,
  "output_dir": "data/measurements",
  "output_format": "parquet",
  "session_name": "example_001"
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `parameters` | array of string | yes | ParameterDB names to record |
| `hz` | number | no | Sampling rate in Hz, valid range `1` to `150` |
| `output_dir` | string | no | Directory for output files |
| `output_format` | string | no | One of `parquet`, `csv`, `jsonl` |
| `session_name` | string | no | Output file stem; autogenerated when omitted |

### Loadstep Request

```json
{
  "duration_seconds": 30.0,
  "loadstep_name": "steady_state",
  "parameters": ["brewcan.temperature.0", "psu.meas_voltage"]
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `duration_seconds` | number | no | Averaging window length in seconds |
| `loadstep_name` | string | no | Human-readable name; autogenerated when omitted |
| `parameters` | array of string | no | Parameter list to average; defaults to the active measurement parameters |

## Endpoints

### `POST /measurement/setup`

Configures the active measurement session.

**Request body** — Measurement Configuration.

**Response** `200 OK`
```json
{
  "ok": true,
  "session_name": "example_001",
  "parameters": ["brewcan.temperature.0", "psu.meas_voltage"],
  "hz": 10.0,
  "output_format": "parquet",
  "warnings": []
}
```

If ParameterDB is connected but some requested parameters are not currently present in the snapshot, setup still succeeds and those names are reported in `warnings`.

---

### `POST /measurement/start`

Starts recording the configured measurement session.

**Response** `200 OK`
```json
{
  "ok": true,
  "message": "Started recording session: example_001"
}
```

---

### `POST /measurement/stop`

Stops recording, flushes the file writer, finalizes any active loadsteps, and returns the session summary.

**Response** `200 OK`
```json
{
  "ok": true,
  "message": "Recording stopped: example_001",
  "samples_recorded": 91,
  "file": "data/measurements\\example_001.parquet",
  "completed_loadsteps": 0,
  "loadsteps": [],
  "missing_parameters": [],
  "warnings": []
}
```

---

### `POST /loadstep/take`

Starts a timed averaging window while recording is active.

**Request body** — Loadstep Request.

**Response** `200 OK`
```json
{
  "ok": true,
  "loadstep_name": "steady_state",
  "duration_seconds": 30.0,
  "parameters": ["brewcan.temperature.0", "psu.meas_voltage"]
}
```

---

### `GET /health`

Reports whether the service can reach ParameterDB.

**Healthy response** `200 OK`
```json
{
  "status": "healthy",
  "details": {
    "backend_connected": true,
    "recording": false,
    "config": null,
    "samples_recorded": 0,
    "active_loadsteps": [],
    "completed_loadsteps_count": 0,
    "missing_parameters": [],
    "warnings": []
  }
}
```

**Unhealthy response** `200 OK`
```json
{
  "status": "unhealthy",
  "reason": "Backend not connected",
  "details": {
    "backend_connected": false,
    "recording": false,
    "config": null,
    "samples_recorded": 0,
    "active_loadsteps": [],
    "completed_loadsteps_count": 0,
    "missing_parameters": [],
    "warnings": []
  }
}
```

---

## Error Responses

| Status | Meaning |
|---|---|
| `400` | Invalid request, backend not connected, no active measurement, or invalid sample rate |
| `500` | Runtime was not initialized |

Common `400` details include:
- `parameterDB backend not connected`
- `At least one parameter is required`
- `Hz must be between 1 and 150, got X`
- `Measurement not configured. Call setup_measurement first.`
- `No measurement in progress`

## Example Flow

```bash
curl -X POST http://localhost:8769/measurement/setup \
  -H "Content-Type: application/json" \
  -d '{
    "parameters": ["brewcan.temperature.0", "psu.meas_voltage"],
    "hz": 10.0,
    "output_format": "parquet",
    "session_name": "example_001"
  }'

curl -X POST http://localhost:8769/measurement/start

curl -X POST http://localhost:8769/loadstep/take \
  -H "Content-Type: application/json" \
  -d '{"duration_seconds": 30.0, "loadstep_name": "steady_state"}'

curl http://localhost:8769/status

curl -X POST http://localhost:8769/measurement/stop
```
